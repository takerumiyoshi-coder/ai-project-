# AIプロジェクト — たける専用AI実験場

> 営業・提案・業務改善を、AIで「その場で形にする」ための実験場。

---

## 1. このプロジェクトの目的

### なぜ作っているか

営業現場で「AIを使うと、その場で形にできる」という体験を
自分の手で作れるようになるため。

ツールを使うだけでなく、仕組みを理解して、
将来的に他の人にも教えられるレベルを目指している。

### 何を目指しているか

- **一旦のゴール**：営業・提案の場で、AIを使って即座にアウトプットを出せる状態
- **その先のゴール**：Claude Codeの仕組みを深く理解し、人に教えられるようになること

### AIをどう使いたいか

「AIに何でもやらせる」のではなく、**役割を分けて組織として動かす**。

調べる・考える・伝える・作る・管理するを、それぞれ専門のAIに担当させる。
人間（たける）は最終判断と方向づけだけを持つ。

## 4. 制作フロー

```
たける（指示・最終判断）
        ↓
ハサウェイ（目的整理・作戦立案・担当振り分け）
        ↓
  ┌─────────────────┐
  イラム             ギギ
  （分析・リサーチ）   （たけるの行動管理）
  └────────┬────────┘
           ↓
        ケネス
  （分析を言葉・構成に変換）
           ↓
        アナハイム
  （LP・資料・デモを制作）
           ↓
    たける（確認・公開判断）
```

- **ゴールが決まる前に制作に入らない**
- **リサーチが終わる前に制作に入らない**
- **各役割の境界を越えない**

## 9. このプロジェクトで大事にしている考え方

### AIに何でもやらせない

AIは万能ではない。1つのAIに全部を任せると、判断がぶれる。
だから「調べる」「作る」「管理する」「伝える」「考える」を分ける。
役割ごとに専門のAIを用意し、人間はゴールと最終判断だけを持つ。

### AIを「組織」として動かす

チームのように役割分担させることで、
精度・速度・再現性が上がる。
ハサウェイが作戦を立て、イラムが調べ、
ケネスが言葉を整え、アナハイムが形にし、
ギギがたけるの動きを整える。

### 実務で使えることを重視する

綺麗に設計することが目的ではない。
営業の場で「その場で形にできる体験」を作れること、
実際に動いて、相手に伝わることが優先。

### 「分かりやすい」より「伝わる」を優先する

情報量が多い説明は伝わらない。
削ること、順番を変えること、相手の温度に合わせることが大事。
AIが作ったものをそのまま使わず、
ケネスが人に届く形へ変換してからアナハイムが制作する。

### 仮説と事実を混ぜない

営業の失敗の多くは「分かったつもり」から始まる。
イラムは事実・仮説・不明を必ず分けて整理する。
根拠のない断定はしない。情報が少ないこと自体を情報として扱う。

### 最終判断は人間が持つ

AIはあくまでたけるの判断を支える存在。
作戦・分析・制作はAIが担うが、
「進める・止める・変える」の判断は常にたけるが行う。
