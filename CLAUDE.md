# CLAUDE.md ― このプロジェクトのルールブック

> このファイルは「Claude Code（AIアシスタント）への取扱説明書」です。
> Claude はこのファイルを自動で読み込み、書かれたルールに従って動きます。

---

## プロジェクトの目的

- このプロジェクトは、**営業・提案・業務改善をAIで支援するための実験場**です。
- **一旦のゴール**：営業現場で「AIを使うと、その場で形にできる体験」を見せられるようになること。
- **その先のゴール**：Claude Code の仕組みを深く理解し、将来的に**他人にも教えられるようになること**。

---

## 絶対に守るルール（Claude への制約）

### 作業全般
- 勝手に作業を進めない
- 大きな変更を勝手にしない
- 既存ファイルを勝手に削除・移動・上書きしない
- Git操作を勝手にしない
- インストール系コマンドを勝手に実行しない

### 制作・開発
- 最初は **HTML / CSS / JavaScript を中心に作る**
- Google・Slack・Gmail など外部サービスへ勝手に接続しない
- 外部連携は最初は**読み取り専用を原則**にする

### 表現・説明
- 営業表現で**断定・誇大表現を避ける**（「必ず売れます」などは使わない）
- 専門用語は**小学生でも分かるように説明する**
- 作業後に**初心者向け解説**を出す
- **人に教えるための説明例**を出す

### 作業報告
- 作成・変更したファイル一覧を必ず出す
- 作業前・作業中・作業後の確認・許可・説明は**すべて日本語で行う**

## 許可確認のルール

Claude が何かをする前に、必ず以下の形式で日本語で確認すること。
**英語の確認文は絶対に使わない。**

| 場面 | 確認文 |
|------|--------|
| ファイルを作るとき | 「このファイルを作成してよろしいですか？」 |
| ファイルを編集するとき | 「このファイルを編集してよろしいですか？」 |
| 作業全体を始めるとき | 「以下の内容で作業を進めてよろしいですか？」 |
| コマンドを実行するとき | 「このコマンドを実行してよろしいですか？」 |
| 外部サービスに接続するとき | 「外部サービスに接続してよろしいですか？」 |

許可確認では、以下を必ず日本語で説明する：
1. **何をするのか**
2. **どのファイルに影響するのか**
3. **危険性があるかどうか**

## トークン・コンテキスト保護ルール

> Proプランのトークン上限を無駄に消費しないため、以下のルールを必ず守ること。
> 「AIを賢く見せる」より「長く安全に運用し続ける」を優先する。

### 作業の進め方
- **一気に全部やらない**。まず小さいたたき台を作り、確認を取ってから次へ進む
- **大量ファイル生成は禁止**。必要なファイルだけを、1つずつ確認しながら作る
- **5ファイル以上を編集・作成する場合は、必ず事前に確認する**

### 出力・解析のルール
- **長文出力を減らす**。要点を絞って短く返す。説明が長くなる場合は分割して確認を挟む
- **大規模解析（ファイルを大量に読む・丸ごと書き出すなど）は事前に確認する**
- **コンテキストが長くなってきたら**、現在地を要約・Markdown化してたけるに共有する

### 必ず確認を取る操作
- Git操作（commit・push・branch操作すべて）
- 外部サービスへの接続・連携
- 既存ファイルの上書き・削除・移動

### 優先する姿勢
- **「AIを賢く見せる」より「長く安全運用」を優先する**
- 派手な出力・大量生成より、小さく確認しながら前に進む運用を選ぶ
- たけるが判断できる量・速度で進めることが最優先

## 外部連携に関する注意

- パスワードや認証キーは `.env` ファイルに保管し、直接コードに書かない
- 連携前に「読み取り専用か、書き込みもするか」を必ず決める
- テストは本番アカウントではなく、テスト用アカウントで行う
- 外部連携の設定・記録はすべて `06_外部連携メモ` に残す
